# 设计说明

`obsidian_llm_wiki_plus` 是一个全新的 Obsidian + LLM Wiki + Agent Skills 双语模板项目。

它不是普通的 Obsidian 目录模板，也不是单纯的任务管理系统。它的目标是把“原始资料、研究分析、结构化知识、项目推进、内容创作、决策记录”放进同一个可演化的 Vault 中，并让 Claude Code、Codex、Gemini CLI 等 Agent 可以按规则协同维护。

## 设计目标

本项目第一版围绕以下目标设计：

1. **资料可追溯**：外部链接、GitHub、PDF、本地文件、网页、论文、视频、粘贴文本等资料进入 `60_原始资料/`，并通过 source card 保存来源信息。
2. **研究可复用**：深度研究结果进入 `30_研究/`，其中长期有价值的实体、概念、观点、方法论、决策可以继续沉淀到 `40_知识库/`。
3. **知识可演化**：知识页需要保留来源、状态、置信度、更新时间、待验证问题和复查周期，避免旧结论长期误导后续判断。
4. **项目可推进**：项目不只是任务清单，还应该记录目标、背景、阶段计划、关键决策、风险和复盘。
5. **内容可生产**：X、公众号、小红书、视频脚本等内容创作应从已有研究、知识页、问答沉淀和决策记录中生长出来。
6. **决策可审计**：技术选型、架构决策、产品判断、内容策略和项目路线需要进入 `80_决策中心/`，便于后续复盘和修正。

## 核心分层模型

本项目使用六层知识操作模型：

```text
资料层：60_原始资料/
研究层：30_研究/、35_问答沉淀/
知识层：40_知识库/
项目层：20_项目/、90_计划/
内容层：70_内容创作/
决策层：80_决策中心/
```

这六层不是孤立目录，而是一条长期工作流：

```text
外部资料 → 资料捕获 → 深度研究 → 知识沉淀 → 项目推进 / 内容创作 / 决策记录 → 复查与演化
```

## 与 llm_wiki 的关系

本项目受 llm_wiki 的思想启发，尤其是：

- 原始资料和加工知识分离。
- 通过 schema 约束知识结构。
- 由 LLM 协助维护持续演化的 Wiki。
- 让资料被“编译”为可复用知识，而不是每次查询时临时拼接答案。

但本项目不是 llm_wiki 的直接复刻。它面向 Obsidian Vault 的实际使用场景，增加了项目推进、内容创作、每日计划、决策记录和多 Agent Skill 路由。

## 与 OrbitOS-style 项目的关系

本项目参考了 OrbitOS 这类 Vault-first 项目的组织方式：

- CN / EN 双版本目录。
- 复制某个语言目录即可作为 Obsidian Vault 使用。
- 在 Vault 目录中通过 Claude Code、Codex、Gemini CLI 等 Agent 工作。
- 使用项目级入口文件和命令/Skill 驱动工作流。

但本项目是全新项目，不是 OrbitOS fork。它的核心模型是“Obsidian + LLM Wiki + Agent Skills + 决策/内容/项目扩展”。

## 为什么需要 Skill 系统

如果所有规则都写进 `CLAUDE.md`、`AGENTS.md` 或 `GEMINI.md`，入口文件会变得过长，Agent 也不容易稳定遵循。

因此本项目采用分层规则：

```text
顶层入口文件：只写核心规则、目录路由和 Skill 路由。
.agents/skills/：放具体任务的可执行规则。
99_系统/：放系统级治理规则、模板、日志和生命周期说明。
.claude/.gemini/.codex：只做命令转发，不重复维护 Skill 内容。
```

这样既能保持入口文件轻量，又能让复杂规则按需加载。

## 设计边界

本项目不是：

- 一个完整应用程序。
- 一个云端知识库产品。
- 一个自动 RAG 系统。
- 一个替代 Obsidian 的工具。
- 一个只适配单一 AI 工具的框架。

本项目是：

- 一个 Obsidian Vault 模板。
- 一个 LLM Wiki 知识结构。
- 一个 Agent Skills 工作流集合。
- 一个适合长期维护的个人 / 团队知识操作系统骨架。

## 第一版优先级

第一版优先保证以下内容可用：

1. CN / EN 双语 Vault 骨架完整。
2. CN / EN Agent 入口文件可用。
3. 7 个核心 Skill 可执行。
4. 系统规则和模板能够支撑资料捕获、研究、知识沉淀、项目、复盘、决策和内容创作。
5. 用户复制 Vault 后能通过 `START_HERE.md` 快速上手。
