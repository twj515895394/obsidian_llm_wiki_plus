# Content Creation Skill

## Goal

Use this skill to create content from the vault’s research, wiki pages, Q&A, project lessons, and decision records.

Content should not start from zero when reusable knowledge already exists.

## Triggers

Use this skill when the user asks for:

- X post
- X thread
- newsletter article
- Rednote post
- video script
- topic library
- content brief
- content calendar
- hot-topic morning/evening brief
- cover or visual planning notes

## Workflow

1. Clarify target platform and audience.
2. Identify the content goal: inform, persuade, explain, sell, debate, or document.
3. Search related material in:
   - `30_Research/`
   - `35_QA_Library/`
   - `40_Knowledge_Base/`
   - `80_Decision_Center/`
4. Extract usable points, stories, claims, and examples.
5. Create a content brief.
6. Draft the content in the platform’s style.
7. Mark possible source links or knowledge references.
8. Save the output into the correct `70_Content_Creation/` subfolder when persistence is needed.
9. If the content produces reusable insights, send them back through `integrate`.

## Output Location

```text
70_Content_Creation/X/
70_Content_Creation/newsletter/
70_Content_Creation/rednote/
70_Content_Creation/video-scripts/
70_Content_Creation/topic-library/
70_Content_Creation/asset-library/
```

## Template

Use:

```text
99_System/templates/content-brief.md
```

## Platform Notes

- X posts should be concise and opinionated.
- X threads should have a strong first post and clear progression.
- Newsletter articles should have structure, examples, and reader value.
- Rednote posts should be direct, visual, and easy to save.
- Video scripts should include scene, narration, pacing, and visual notes.

## Boundaries

Do not fabricate sources.

Do not create generic AI-style content when the vault contains specific experience or judgment.

Do not mix draft output with raw sources.

## Quality Check

Before finishing, check:

- Audience is clear.
- Platform format is respected.
- Core idea is sharp.
- Knowledge sources are reusable and linked when possible.
- The draft has a strong opening.
- Next action is clear: publish, revise, save, or integrate.
