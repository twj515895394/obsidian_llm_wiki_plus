# AI 知识库是否需要原始资料层

## 决策问题

`obsidian_llm_wiki_plus` 第一版是否需要单独设计 `60_原始资料/`，而不是只保留研究笔记和知识页？

## 背景

用户经常会把 GitHub 链接、技术文章、PDF、产品文档、长文本资料交给 Agent 分析。如果系统只保存分析结果，后续很难判断结论是否可靠，也无法处理资料更新、冲突或过期问题。

## 备选方案

| 方案 | 优点 | 缺点 |
|---|---|---|
| 不设原始资料层 | 目录更简单 | 结论不可追溯 |
| 只保存链接 | 轻量 | 链接失效后无法复查 |
| 分级保存原始资料 | 平衡可追溯与成本 | 需要 capture 规则 |

## 最终决策

采用“分级保存原始资料”的方案，并在 `capture` Skill 中要求 Agent 主动判断资料等级，必要时询问用户是否保存 source card。

## 决策依据

- 技术和 AI 工具信息变化快，需要可复查来源。
- 研究、知识页和决策记录都需要证据链。
- 分级保存可以避免把 Vault 变成资料垃圾场。

## 风险与复查

- 风险：用户可能觉得保存确认过于频繁。
- 缓解：只对 S/A 级资料主动建议保存。
- 复查时间：2026-08-24。
