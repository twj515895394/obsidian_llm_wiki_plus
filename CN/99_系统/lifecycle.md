# lifecycle.md

## 目标

定义知识、资料、研究、决策和内容的生命周期，防止知识库长期失控或过期。

## 生命周期状态

| 状态 | 含义 |
|---|---|
| draft | 草稿，未完成 |
| active | 当前有效 |
| needs-review | 需要复查 |
| stale | 可能过期 |
| deprecated | 已废弃，不建议使用 |
| archived | 已归档，仅保留历史 |

## 适用对象

- source card
- 研究笔记
- 知识页
- 决策记录
- 项目文档
- 内容草稿

## 复查建议

| 内容类型 | 建议复查周期 |
|---|---|
| AI 工具能力判断 | 30-90 天 |
| 技术选型 | 3-6 个月 |
| 架构决策 | 6-12 个月 |
| 内容策略 | 1-3 个月 |
| 项目计划 | 每周或每阶段 |
| 原始资料链接 | 3-12 个月 |

## 过期处理

发现内容可能过期时：

1. 不要直接删除。
2. 标记 `status: stale` 或 `needs-review`。
3. 写明过期原因。
4. 放入 `90_计划/待审核/`。
5. 用户确认后再归档或更新。

## 废弃处理

废弃内容应保留历史原因：

```markdown
## 废弃原因

## 被什么替代

## 废弃日期

## 相关决策
```

## 归档原则

归档不是删除。归档内容应移动到对应 `归档/` 目录，并尽量保留来源链接和上下文。
