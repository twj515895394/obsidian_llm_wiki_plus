# ingest.md

## 目标

定义外部资料进入 Vault 的流程，保证资料可追溯、可复查、可二次加工。

## 入口

资料可能来自：

- URL
- GitHub 仓库
- 本地文件路径
- 上传文件
- PDF / Word / Excel
- 图片 / 音频 / 视频
- 网页文章
- 论文
- 官方文档
- API 文档
- 用户粘贴的长文本

## 标准流程

1. 触发 `capture` Skill。
2. 判断资料等级 S/A/B/C。
3. 询问用户是否保存。
4. 用户确认后创建 source card。
5. 必要时保存附件、快照或导入内容。
6. 更新 `60_原始资料/source-index.md`。
7. 如果需要分析，继续触发 `research`。
8. 如果分析结果可复用，继续触发 `integrate`。

## 资料等级

| 等级 | 保存方式 |
|---|---|
| S | source card + 尽量完整副本 / 快照 |
| A | source card + 摘要 / 关键片段 |
| B | source card 或轻量链接记录 |
| C | 默认不保存 |

## 原始资料原则

- 原始资料不得被 AI 总结覆盖。
- AI 摘要只能作为 source card 的附加字段。
- 如果没有保存完整副本，必须保留原始链接和访问日期。
- 如果资料可能变化，应尽量保存快照或版本说明。

## 命名建议

```text
YYYY-MM-DD_资料标题.md
```

## 后续处理

- 保存资料后，不默认进入研究。
- 用户要求分析时，使用 `research`。
- 用户要求沉淀时，使用 `integrate`。
- 用户基于资料做选择时，使用 `decision-record`。
