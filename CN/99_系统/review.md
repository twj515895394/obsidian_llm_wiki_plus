# review.md

## 目标

定义哪些情况需要人工审核，避免 Agent 自动污染知识库。

## 必须进入待审核的情况

以下情况不要擅自处理，应记录到：

```text
90_计划/待审核/
```

- 新资料与旧结论冲突。
- 需要删除知识页。
- 需要合并多个知识页。
- 需要修改重要决策记录。
- 资料可信度不明但会影响结论。
- 某个技术判断可能过期。
- 批量迁移、归档或重命名文件。
- 涉及隐私、公司资料或敏感信息。

## 审核事项格式

```markdown
# 审核事项：标题

## 类型

冲突 / 合并 / 删除 / 过期 / 决策变更 / 资料可信度 / 其他

## 背景

## 涉及文件

## Agent 建议

## 需要用户确认的问题

## 推荐答案

## 截止时间 / 复查时间
```

## 审核原则

- Agent 可以提出建议，但不要擅自做高影响修改。
- 每次只问用户一个关键问题。
- 问题必须附带推荐答案。
- 用户确认后，再执行变更。

## 适用 Skill

- `integrate`：页面合并、冲突处理、知识过期。
- `decision-record`：重要决策变更。
- `capture`：敏感资料保存。
- `kickoff`：项目结构批量创建。
