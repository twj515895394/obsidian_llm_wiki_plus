# 知识沉淀 Skill

## 目标

将研究笔记、问答沉淀、项目经验、资料分析、决策过程中的可复用内容，整理进 `40_知识库/`，形成结构化 Wiki。

本 Skill 的核心不是“生成更多笔记”，而是维护一个可追溯、可复用、可演化的知识网络。

## 触发条件

用户出现以下意图时使用本 Skill：

- “整理成知识库”
- “沉淀为 Wiki”
- “提取概念”
- “生成原子笔记”
- “更新知识页”
- “把这次讨论沉淀一下”
- “把研究结果整理成长期知识”
- “把这段文本结构化”
- “从这份资料里提取实体 / 概念 / 方法论 / 观点”

## 知识类型路由

| 类型 | CN 目录 | 说明 |
|---|---|---|
| 实体 | `40_知识库/实体/` | 人物、公司、产品、项目、工具、组织 |
| 概念 | `40_知识库/概念/` | 技术概念、理论、术语、模型 |
| 方法论 | `40_知识库/方法论/` | SOP、流程、最佳实践、操作方法 |
| 观点 | `40_知识库/观点/` | 可验证判断、主张、结论、趋势判断 |
| 决策 | `40_知识库/决策/` | 与 `80_决策中心/` 相关的知识化摘要 |
| 综合分析 | `40_知识库/综合分析/` | 跨多个来源的综合理解 |
| 对比分析 | `40_知识库/对比分析/` | 多对象、多方案、多工具对比 |

## 执行流程

1. 读取输入内容
   - 研究笔记
   - 问答记录
   - 项目文档
   - source card
   - 决策记录
   - 用户粘贴内容

2. 提取候选知识
   - 实体
   - 概念
   - 方法论
   - 观点
   - 决策
   - 对比关系
   - 综合判断

3. 检查已有页面
   - 优先更新已有页面。
   - 避免创建重复概念。
   - 如果存在相似页面，应建议合并或放入待审核。

4. 创建或更新知识页
   - 使用 `99_系统/模板/wiki-page.md`。
   - 保留来源、状态、置信度、更新时间。

5. 建立链接
   - 链接到来源 source card。
   - 链接到相关研究笔记。
   - 链接到相关实体、概念、方法论、决策。

6. 更新索引
   - 更新 `40_知识库/index.md`。
   - 如有整体判断变化，更新 `40_知识库/overview.md`。

## 非结构化文本解析模式

当用户提供一大段未整理文本、聊天记录、会议纪要、调研摘录或散乱笔记，并希望“整理 / 拆解 / 入库”时，使用本模式。

### 处理步骤

1. 判断输入是否应该先进入 `60_原始资料/`。
   - 如果是外部资料或可追溯来源，先触发 `capture`。
   - 如果只是用户自己的临时整理，可直接进入本模式。

2. 识别主主题。
   - 提取一个主 Topic。
   - 判断它更适合进入 `30_研究/` 还是直接更新已有知识页。

3. 提取原子知识候选。
   - 实体：人、公司、产品、项目、工具。
   - 概念：术语、模型、理论、机制。
   - 方法论：步骤、流程、SOP、实践经验。
   - 观点：判断、结论、趋势、假设。
   - 对比：多个对象之间的差异。

4. 生成或更新主笔记。
   - 如果输入包含完整分析过程，保存到 `30_研究/`。
   - 如果只是若干可复用知识点，直接进入 `40_知识库/`。

5. 生成或更新原子知识页。
   - 优先更新已有页面。
   - 新页面必须有清晰定义、来源、当前判断、待验证问题。

6. 建立双向链接。
   - 主笔记链接到原子知识页。
   - 原子知识页链接回主笔记或 source card。

### 输出报告格式

```markdown
## 知识整理完成

**主笔记:** [[主题]]

**已更新 / 创建知识条目:**
- [[概念A]]：一句话说明
- [[实体B]]：一句话说明
- [[方法论C]]：一句话说明

**关联关系:**
- 主笔记链接到 N 个知识条目
- 建立了 M 个跨页面链接

**待审核:**
- [ ] 需要确认的问题
```

## 冲突处理

出现以下情况，不要直接覆盖：

- 新资料和旧结论冲突
- 两个页面概念高度重复
- 某个观点可能已经过期
- 用户此前做过相反决策
- 需要删除或合并页面

处理方式：

1. 在页面中标记冲突。
2. 在 `90_计划/待审核/` 生成审核事项。
3. 等用户确认后再合并、删除或改变结论。

## 边界规则

- 不要把所有内容都拆成原子笔记。
- 不要为了“完整”创建低价值页面。
- 不要把临时想法直接提升为知识页。
- 强判断必须有来源或明确标记为“个人判断”。
- 决策记录的完整版本应保存在 `80_决策中心/`，知识库中只保留知识化摘要。
- `50_资源/` 只放资源入口、工具、链接、案例和提示词，不放结构化知识页。

## 质量检查

完成后检查：

- 是否优先检查了已有页面？
- 是否正确分类到知识类型？
- 是否保留来源？
- 是否建立双向链接？
- 是否更新 index / overview？
- 是否标记置信度和复查时间？
- 是否把冲突事项放入待审核？
