# 决策记录 Skill

## 目标

记录重要判断的背景、备选方案、依据、风险和复查时间，让决策可追溯、可审计、可复盘。

本 Skill 适合记录“为什么当时选择了这个方案”，而不是只记录结论。

## 触发条件

用户讨论以下内容时，应考虑使用本 Skill：

- 技术选型
- 架构决策
- 产品判断
- 内容策略
- 项目路线
- 是否采用某工具
- 是否启动某项目
- 是否放弃某方向
- 团队流程或规范变更
- 重要资源投入选择

## 决策分类

| 内容策略 | `80_决策中心/内容策略/` |

## 执行流程

1. 识别决策问题
   - 到底要决定什么？
   - 决策影响范围是什么？

2. 补齐背景
   - 为什么现在要做这个决策？
   - 有哪些约束？
   - 与哪些项目、资料、研究相关？

3. 列出备选方案
   - 方案 A
   - 方案 B
   - 方案 C
   - 暂不行动

4. 对比方案
   - 成本
   - 收益
   - 风险
   - 可逆性
   - 实施难度
   - 对长期知识库的影响

5. 形成决策
   - 最终选择
   - 推荐理由
   - 不选择其他方案的原因

6. 设置复查
   - 什么时间复查？
   - 触发什么条件需要重新评估？

7. 保存记录
   - 使用 `99_系统/模板/decision-record.md`。
   - 更新 `80_决策中心/decision-index.md`。

## 输出位置

默认：

```text
80_决策中心/
```

推荐命名：

```text
YYYY-MM-DD_决策主题.md
```

## 决策记录结构

```markdown
# 决策主题

## 决策摘要

## 背景

## 决策问题

## 备选方案

## 对比分析

## 最终决策

## 决策依据

## 风险与代价

## 执行计划

## 复查条件

## 相关资料

## 更新记录
```

## 边界规则

- 不要把普通建议都升级为决策记录。
- 没有明确选择时，可以记录为“待决策”。
- 高风险决策需要进入 `90_计划/待审核/`。
- 决策依据应引用研究笔记、source card 或已有知识页。
- 如果决策后形成通用原则，应使用 `integrate` 沉淀到 `40_知识库/`。

## 质量检查

完成后检查：

- 决策问题是否明确？
- 是否列出备选方案？
- 是否写清为什么选择当前方案？
- 是否记录风险和代价？
- 是否设置复查时间或触发条件？
- 是否更新 decision-index？
