# 轻量问答 Skill

## 目标

用于快速回答用户问题，避免所有问题都进入重型的 capture、research 或 integrate 流程。

本 Skill 的核心是：先把问题回答清楚；只有当答案具有复用价值时，才建议沉淀到 Vault。

## 触发条件

用户提出以下类型问题时使用本 Skill：

- 简单概念解释
- 命令、报错、配置含义说明
- 目录应该放哪里
- 某个工具或术语的快速判断
- 对已有 Vault 内容的快速问答
- 用户明确使用 `/ask` 或说“简单回答一下”

## 执行流程

1. 判断问题是否需要查询 Vault。
   - 如果与已有项目、研究、知识页有关，可快速查找 `30_研究/`、`35_问答沉淀/`、`40_知识库/`、`80_决策中心/`。
   - 如果只是常识性或临时问题，可直接回答。

2. 直接回答问题。
   - 优先给出清晰结论。
   - 必要时补充简短理由、例子或代码片段。
   - 如果引用已有笔记，使用 Obsidian wikilink。

3. 判断是否值得沉淀。
   - 普通临时问题：不保存。
   - 高价值问答：建议保存到 `35_问答沉淀/`。
   - 可长期复用的概念、方法或判断：建议后续交给 `integrate`。

## 输出格式

```markdown
[直接回答]

[必要的解释 / 示例]

[可选：相关笔记]

[可选：是否建议保存]
```

## 保存建议话术

```text
这个回答后续可能会复用。我建议保存到 `35_问答沉淀/`，后续也可以通过 `integrate` 沉淀到 `40_知识库/`。是否保存？
```

## 边界规则

- 不要为简单问题创建计划文件。
- 不要为临时问答无脑创建知识页。
- 不要把轻量问答升级成深度研究，除非用户明确需要。
- 如果问题涉及外部资料、链接、PDF、GitHub 或长文本，应考虑切换到 `capture` 或 `research`。
- 如果答案会影响重要决策，应建议使用 `decision-record`。

## 质量检查

完成后检查：

- 是否直接回答了问题？
- 是否避免了过度流程化？
- 是否在必要时引用已有笔记？
- 是否只在有复用价值时建议保存？
