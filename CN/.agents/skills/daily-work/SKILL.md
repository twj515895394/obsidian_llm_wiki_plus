# 每日计划 / 复盘 Skill

## 目标

帮助用户管理每日工作流，包括今日计划、工作记录、推进状态、问题阻塞、每日复盘和明日安排。

本 Skill 的目标不是做普通日记，而是让每日记录成为项目推进、知识沉淀和决策复盘的入口。

## 触发条件

用户出现以下意图时使用本 Skill：

- “开始今天”
- “今日计划”
- “今天要做什么”
- “记录一下今天”
- “每日复盘”
- “总结今天”
- “安排明天”
- “今天推进了这些……”

## 执行模式

### 模式 A：开始一天 / 早间计划

1. 获取今天日期。
2. 读取昨日笔记。
   - 路径：`10_日记/YYYY-MM-DD.md`
   - 提取未完成任务。
   - 记录昨日推进内容。
3. 检查活跃项目。
   - 搜索 `20_项目/` 中 `status: active` 的项目。
   - 识别当前阶段、待办、最近更新时间和可能过期的项目。
4. 检查收件箱。
   - 查看 `00_收件箱/` 中待处理内容。
   - 统计数量，提示是否需要清理。
5. 检查计划区。
   - `90_计划/待执行/`
   - `90_计划/待审核/`
   - `90_计划/路线图/`
6. 询问用户今天主要目标。
   - 优先基于活跃项目给选项。
   - 允许用户输入新目标。
7. 创建或更新今日笔记。
   - 使用 `99_系统/模板/daily-note.md`。
   - 保存到 `10_日记/YYYY-MM-DD.md`。

### 模式 B：日间记录

记录：

- 实际完成
- 临时插入
- 决策变化
- 新资料
- 新问题
- 可沉淀知识

如出现外部资料，触发 `capture`。
如出现重要判断，触发 `decision-record`。
如出现可复用经验，触发 `integrate`。

### 模式 C：晚间复盘

1. 对照今日计划。
2. 总结完成情况。
3. 记录未完成事项。
4. 识别阻塞和原因。
5. 提取可复用经验。
6. 形成明日建议。
7. 必要时更新项目、知识库或决策中心。

## 今日笔记结构

```markdown
# YYYY-MM-DD 每日记录

## 今日重点

## 计划事项

## 实际推进

## 临时事项

## 阻塞与问题

## 新资料

## 新判断 / 决策

## 可沉淀知识

## 今日复盘

## 明日安排
```

## 输出位置

默认：

```text
10_日记/
```

推荐命名：

```text
YYYY-MM-DD.md
```

新想法但未处理时，放入：

```text
00_收件箱/
```

未来任务或待审核事项，放入：

```text
90_计划/待执行/
90_计划/待审核/
```

## 与其他 Skill 的联动

- 简单问题：触发 `ask`。
- 出现外部资料：触发 `capture`。
- 出现研究问题：触发 `research`。
- 出现可复用经验：触发 `integrate`。
- 出现重要判断：触发 `decision-record`。
- 出现新项目：触发 `kickoff`。
- 完成阶段收尾：触发 `archive`。

## 边界规则

- 不要把每日计划写得过满。
- 今日重点最多 3 个。
- 不要把所有临时想法都沉淀成知识页。
- 复盘要区分事实、感受、问题和下一步。
- 如果一天没有完整记录，也可以做轻量复盘。
- 如果今日笔记已存在，不要覆盖，应该合并更新。

## 质量检查

完成后检查：

- 今日重点是否少而清晰？
- 是否读取了昨日笔记和未完成任务？
- 是否检查了活跃项目和收件箱？
- 是否记录实际推进？
- 是否识别阻塞？
- 是否形成明日安排？
- 是否有需要进入项目、知识库或决策中心的内容？
