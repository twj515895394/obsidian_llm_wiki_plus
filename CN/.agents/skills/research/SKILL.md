# 深度研究 Skill

## 目标

围绕一个项目、技术、产品、行业、工具、问题或主题进行系统研究，输出可复查的研究笔记，并标记后续可沉淀到知识库的内容。

本 Skill 负责“研究分析”，不直接承担长期 Wiki 入库。研究完成后，如有可复用知识，应继续使用 `integrate`。

## 触发条件

用户出现以下意图时使用本 Skill：

- “分析这个项目 / 文档 / 技术 / 产品”
- “帮我调研……”
- “对比 A 和 B”
- “读一下这个资料并总结”
- “这个方案是否可行”
- “帮我形成一份研究报告”
- “结合已有资料给我判断”

## 前置检查

研究前先检查：

1. 是否存在外部资料需要先用 `capture` 保存？
2. `30_研究/` 是否已有相关研究笔记？
3. `40_知识库/` 是否已有相关概念、实体、观点、决策？
4. `80_决策中心/` 是否已有相关决策记录？
5. 本次研究是否需要用户确认研究范围？

如果资料来源不稳定或需要联网确认，应明确说明来源和不确定性。

## 执行流程

1. 明确研究问题
   - 研究对象是什么？
   - 用户真正想判断什么？
   - 输出是报告、对比、建议，还是可执行方案？

2. 盘点已有上下文
   - 检查已有研究、知识页、决策记录、source card。
   - 避免重复生成已有内容。

3. 建立研究框架
   - 背景
   - 核心问题
   - 资料来源
   - 分析维度
   - 关键发现
   - 风险与限制
   - 可沉淀知识
   - 后续行动

4. 生成研究笔记
   - 默认保存到 `30_研究/`。
   - 使用 `99_系统/模板/research-note.md`。

5. 标记后续沉淀
   - 可进入 `40_知识库/概念/` 的概念
   - 可进入 `40_知识库/实体/` 的项目/产品/工具
   - 可进入 `40_知识库/观点/` 的判断
   - 可进入 `40_知识库/方法论/` 的流程和 SOP
   - 可进入 `80_决策中心/` 的重要选择

## 输出位置

默认：

```text
30_研究/
```

推荐命名：

```text
YYYY-MM-DD_研究主题.md
```

示例：

```text
30_研究/2026-05-23_OrbitOS与llm_wiki设计对比.md
```

## 研究笔记结构

```markdown
# 研究主题

## 研究问题

## 背景

## 资料来源

## 核心发现

## 分析过程

## 结论

## 风险与限制

## 可沉淀知识

## 后续行动
```

## 边界规则

- 研究不是资料保存；原始资料必须引用 `60_原始资料/`。
- 研究不是最终决策；重要判断需要进入 `decision-record`。
- 研究不是知识入库；可复用内容需要交给 `integrate`。
- 如果研究涉及实时信息、政策、价格、版本、新闻，应标记研究日期和来源。
- 如果资料相互冲突，应单独列出“冲突信息”。

## 质量检查

完成后检查：

- 研究问题是否清楚？
- 是否引用了来源？
- 是否区分事实、分析和判断？
- 是否写出了风险与限制？
- 是否标记了可沉淀知识？
- 是否需要生成决策记录？
- 是否需要继续执行 `integrate`？
