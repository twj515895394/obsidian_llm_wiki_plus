# 内容创作 Skill

## 目标

从已有资料、研究笔记、知识页、问答沉淀和决策记录中生成可发布或可继续打磨的内容草稿。

本 Skill 的核心不是“凭空写内容”，而是让内容创作从知识库中生长出来。

## 触发条件

用户要求生成以下内容时使用本 Skill：

- X 推文
- X Thread
- X Article
- 公众号文章
- 小红书笔记
- 视频脚本
- 选题库
- 热点早晚报
- 配图清单
- 内容大纲
- 发布计划

## 执行流程

1. 明确平台
   - X
   - 公众号
   - 小红书
   - 视频
   - Newsletter
   - 其他

2. 明确目标
   - 科普
   - 观点表达
   - 经验分享
   - 技术分析
   - 营销传播
   - 复盘总结

3. 明确受众
   - 技术开发者
   - 普通 AI 用户
   - 团队管理者
   - 内容创作者
   - 业务人员

4. 提取素材
   优先从以下目录读取：
   - `30_研究/`
   - `35_问答沉淀/`
   - `40_知识库/`
   - `80_决策中心/`
   - `60_原始资料/`

5. 生成结构
   - 标题 / Hook
   - 核心观点
   - 论证路径
   - 案例 / 类比
   - 结尾行动建议

6. 输出草稿
   - 保存到 `70_内容创作/` 对应平台目录。
   - 使用 `99_系统/模板/content-brief.md`。

7. 可选回流
   - 内容创作过程中产生的新观点，可回流到 `40_知识库/观点/`。
   - 形成的新方法，可回流到 `40_知识库/方法论/`。
   - 重要策略选择，可进入 `80_决策中心/内容策略/`。

## 输出位置

| 内容类型 | 目录 |
|---|---|
| X 内容 | `70_内容创作/X/` |
| 公众号 | `70_内容创作/公众号/` |
| 小红书 | `70_内容创作/小红书/` |
| 视频脚本 | `70_内容创作/视频脚本/` |
| 选题 | `70_内容创作/选题库/` |
| 素材 | `70_内容创作/素材库/` |

## 内容简报结构

```markdown
# 内容标题

## 平台

## 目标受众

## 核心观点

## 使用素材

## 内容结构

## 正文草稿

## 配图 / 视频建议

## 发布注意事项

## 可回流知识
```

## 风格规则

- 少写空泛套话。
- 优先使用具体场景、具体判断、具体案例。
- 标题可以有吸引力，但不要虚假夸张。
- 技术内容要保证准确，不懂就标记待确认。
- 面向社交平台时，开头必须有明确 Hook。
- 面向公众号时，结构要清楚，有段落节奏。
- 面向小红书时，要注意图文配合和用户共鸣。

## 边界规则

- 不要凭空编造来源、数据或案例。
- 不要把未经验证的强判断包装成事实。
- 如果内容基于外部资料，应保留来源。
- 如果生成图片提示词或视频脚本，应单独标记素材需求。
- 如果内容方向涉及重要策略，应考虑生成 `decision-record`。

## 质量检查

完成后检查：

- 平台和受众是否明确？
- 是否有清晰 Hook？
- 是否复用了知识库素材？
- 是否有核心观点？
- 是否避免 AI 味和空泛表达？
- 是否标记可回流知识？
